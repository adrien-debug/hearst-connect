📊 Flow Analysis: HashVault vs Hearst Connect

Analyse comparative complète des parcours utilisateurs • QA Step-by-Step

🏗️ 1. Architecture Overview

🔵 HashVault (Mockup HTML)
  • Single Page App (SPA)
  • 3 tabs internes: Dashboard, My Vaults, Invest
  • Wallet déjà connecté au chargement
  • Pas de routing réel (JS show/hide)
  • Charts.js pour visualisations
  • Aucune auth gate visible
🟢 Hearst Connect (Next.js)
  • Multi-page application
  • 4 routes distinctes: /, /vaults, /app, /admin
  • Wallet connect progressif
  • Next.js App Router
  • Custom Canvas component
  • Auth gates sur /app et /admin

⚖️ 5. Différences Clés

🔶 Différences UX majeures
  • Onboarding: HashVault = aucun | Hearst = Landing → Vaults → App (progressive disclosure)
  • Wallet: HashVault présume connexion | Hearst gère 3 états (disconnected/wrong chain/connected)
  • Navigation: HashVault = SPA tabs | Hearst = Multi-page routing
  • Invest flow: HashVault = Stepper visuel 4 étapes | Hearst = Modal puis app
  • Data visualization: HashVault = plus riche (scenarios charts, time-to-target) | Hearst = plus simple
  • Portfolio: HashVault = Dashboard centralisé | Hearst = Canvas dans /app protégé
✅ Points forts HashVault (à considérer)
  • Stepper clair avec 4 étapes visuelles (Select/Product/Deposit/Confirmed)
  • Time to target simulation avec 3 scénarios marché (Bull/Sideways/Bear)
  • Dynamic allocation preview interactif (click scenarios pour voir rebalancing)
  • Validation visuelle du montant (green border + checkmark quand valid)
  • Term sheet embedded en base64 (pas de lien externe)
  • Multiple positions du même vault (#1, #2) bien gérées
  • Dashboard → My Vaults deep link (click row table = scroll to vault card)
⚠️ Points faibles HashVault
  • Aucune auth gate: présume wallet connecté (pas réaliste production)
  • Pas de gestion chain: pas de check Base / switch network
  • Mockup statique: toutes les positions hardcodées, pas de fetch réel
  • Pas de landing: pas d'onboarding pour nouveaux utilisateurs
  • SPA limitations: pas de deep links shareables, SEO limité

🚨 7. Gaps Critiques à Adresser

⛔ Gaps majeurs dans Hearst Connect actuel
  • Pas de stepper invest visible: HashVault a un flow clair en 4 étapes, Hearst semble passer direct modal → app
  • Pas de time-to-target simulation: feature clé de HashVault absent (3 scenarios marché)
  • Pas de dynamic allocation preview: HashVault montre rebalancing selon market regime
  • Validation montant moins claire: HashVault a feedback visuel riche (border verte, checkmark)
  • Pas de term sheet inline: HashVault l'embed en base64, Hearst pourrait avoir lien externe
  • Multiple positions même vault: gestion #1, #2, #3 pas claire dans Hearst
🔧 Recommendations
  • Implémenter stepper invest: Select → Product → Deposit → Confirmed
  • Ajouter time-to-target chart: 3 scenarios (Bull/Sideways/Bear) avec target unlock visual
  • Scenarios interactifs: Click pour preview allocation (comme HashVault)
  • Améliorer validation input: Border verte + checkmark quand montant valid
  • Term sheet embedded: PDF en base64 plutôt que lien externe
  • Gestion positions multiples: Permettre #1, #2, #3 pour même vault
  • Dashboard centralisé: Comme HashVault, page récap complète avant deep dive positions
