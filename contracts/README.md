# Hearst Vault Contracts

Cohort-based USDC vault with on-chain rebalancing attestation, deployed behind UUPS proxies on Base. The first product (Hearst Prime) targets RWA mining + USDC yield + BTC-hedged exposure off-chain; this contract is the public ledger of principal, allocation weights, signal-driven rebalances, and yield distributions.

## Layout

```text
contracts/
├── src/
│   ├── HearstVault.sol           # main UUPS implementation (multi-cohort, fees, rebalance)
│   ├── HearstPosition.sol        # ERC-721 soulbound cohort receipt
│   ├── interfaces/               # IHearstVault, IHearstPosition
│   └── types/Enums.sol           # MarketRegime, SignalType
├── test/
│   ├── unit/                     # Deposit, Claim, Withdraw, Rebalance, Fees, Roles, Lifecycle, Position
│   └── helpers/                  # VaultFixture, MockUSDC
├── script/Deploy.s.sol           # Base Sepolia + mainnet deploy with role assertions
└── foundry.toml
```

## Setup

Foundry deps are pinned but not committed. Install once after cloning:

```sh
npm run contracts:install
```

This pulls forge-std + OpenZeppelin (regular + upgradeable) into `lib/`.

## Build & test

```sh
npm run contracts:build       # compiles + regenerates ABI files in src/config/abi/
npm run contracts:test        # forge test -vvv
npm run contracts:coverage    # forge coverage --report lcov
```

The frontend ABIs in `src/config/abi/hearst-vault-v2.ts` and `hearst-position.ts` are auto-generated by `scripts/sync-abi.mjs` and checked into git.

## Deploy to Base Sepolia

Requires `.env` with `DEPLOYER_PRIVATE_KEY`, `ADMIN_ADDRESS` (Safe multisig), `BASE_SEPOLIA_RPC_URL`, `BASESCAN_API_KEY`.

```sh
npm run contracts:deploy:sepolia
```

The script asserts the deployer no longer holds `DEFAULT_ADMIN_ROLE` after broadcasting. Output addresses land in `src/config/contracts/base-sepolia.json` for the Next.js app to consume.

## Key design choices

- **No swaps, no oracle, no MEV surface** — the vault doesn't move underlying assets; it records intent. Real rebalances happen off-chain at the custodian.
- **MasterChef-style accounting** — `accRewardPerShare` accumulator per vault, `rewardDebt` per cohort NFT.
- **Asymmetric pause** — only `deposit` is `whenNotPaused`. Claims and withdrawals stay open during incidents.
- **Per-cohort cumulative target** — each cohort tracks its own progress to early maturity (default 36% for Prime).
- **Capital recovery extension** — operator can extend a cohort's lockup if principal is underwater at maturity, capped at 730 days per call, additive across calls.

## Security notes for audit

- Roles: `DEFAULT_ADMIN_ROLE` (timelock-protected upgrades + fees), `OPERATOR_ROLE` (rebalance, notifyRewards, extendLock), `KEEPER_ROLE` (advanceEpoch only), `PAUSER_ROLE` (incident response).
- Reentrancy: `nonReentrant` on deposit/claim/withdraw/notifyRewardAmount.
- SafeERC20 + SafeCast everywhere uint128 truncation could matter.
- Position NFT is soulbound v1 — `_update` reverts on transfer (`from != 0 && to != 0`).
- `withdrawFees()` recipient is fixed by `setTreasury` — limits blast radius if an admin key is compromised.
